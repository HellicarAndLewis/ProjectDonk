Detailed Description

Provides a persistent settings interface for stuff not in the GUI.

It's basically a hashmap that saves itself every time you set a value. Also, if you ever try to get a value, you can optionally provide a default so it creates a default settings file if one didn't exist in the first place.

It's also accessible everywhere, just include Settings.h and then do something like

int width = settings.getInt("width", 1024);

Which will load the xml if not done already, then return the width value. If width hasn't been set, or there is no xml file, one will be created, using the default value set in the second parameter.

The file format is xml, so easy on the eye when editing.
